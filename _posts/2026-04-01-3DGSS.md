> **3D Gaussian Splatting（3DGS）** 面向 **新视角合成（Novel View Synthesis）**：给定多视角图像，优化一组 3D 高斯原语，使任意视点下的合成图像与观测一致。相较早期 NeRF 的「沿射线密集查询 MLP」，3DGS 以 **显式场景表示 + GPU 友好光栅化** 著称，在质量与 **实时渲染**（常见 30–100+ FPS 量级，视分辨率与实现而定）之间取得了突出平衡。

下文按「表示 → 投影与合成 → 优化 → 与 NeRF 对比 → 机器人侧应用 → 局限」组织，便于与 NeRF、网格、点云等路线对照阅读。

## 1. 直观比喻与问题设定

若用一句话类比：场景由大量 **半透明椭球状「墨点」** 在空间中叠加而成；每个墨点有位置、朝向、尺度、不透明度和视角相关颜色，整体经投影与排序后混成图像。与 **多边形网格** 的硬边表面不同，3DGS 更贴近 **体积/软占据** 的显式参数化，但实现上针对现代 GPU 做了高度特化。

## 2. 单个 3D 高斯：参数化

每个高斯原语可理解为 3D 空间中的 **各向异性高斯核**，常用参数包括：

| 参数 | 记号 / 含义 |
| :--- | :--- |
| **中心** | 位置 $\mu \in \mathbb{R}^3$ |
| **协方差 / 形状** | 由 **缩放** 与 **旋转** 参数化（实践中常分解为 $\mathbf{R}\mathbf{S}$ 等稳定形式，避免直接优化非法协方差矩阵） |
| **不透明度** | $\alpha$，控制该原语对合成的贡献 |
| **颜色** | 常用 **球谐函数（SH）** 系数描述 **视角相关** 外观（高光、非朗伯反射等） |

颜色用 SH 扩展，是为了在固定原语个数下，仍能近似 **与视线方向有关的辐射**，这是与「每点仅 RGB」的重要区别。

## 3. Splatting：从 3D 到 2D

**Splatting** 在图形学中由来已久：在 3DGS 里指将每个 3D 高斯 **投影** 到当前相机图像平面，得到 2D 高斯 footprint（可视作屏幕空间椭圆），再与深度排序和 **alpha 合成** 结合，得到整幅渲染图。

要点：

1. **投影**：由当前内外参将 3D 高斯映射到图像平面（论文中给出与雅可比相关的近似，使整条管线对参数可微）。
2. **排序与混合**：按深度从前往后（或约定一致顺序）做 $\alpha$-blending，与体渲染的「沿射线积分」在形式不同，但目标仍是解释观测颜色。

因此，**一次前向渲染**主要是大量 2D 高斯 tile 的光栅化与混合，而非 NeRF 中逐射线数百次 MLP 查询，这是实时性的主要来源之一。

## 4. 优化与自适应密度控制

典型训练闭环：

1. **初始化**：常用 **SfM**（如 COLMAP）得到稀疏点云，每个点对应一个初始小高斯。
2. **可微渲染**：当前参数下 splat 出图像，与真实照片算光度损失（可加 SSIM、感知损失等）。
3. **反向传播**：更新各高斯的 $\mu$、协方差相关参数、$\alpha$、SH 系数等。

**自适应密度控制**（原论文核心工程机制之一）在优化过程中周期性调整原语集合规模：

- **Clone**：小高斯落在高梯度区域时，可复制以增加局部容量。
- **Split**：过大高斯跨物体边界时，可分裂为更小原语以贴合细节。
- **Prune**：不透明度过低或无效的大原语可剔除，抑制冗余。

迭代后，原语往往 **沿表面与高频细节处** 聚集，从而在有限个数下拟合复杂场景。

## 6. 与 NeRF 的对比（何时选谁）

| 维度 | NeRF 类（隐式 MLP / 体渲染） | 3DGS |
| :--- | :--- | :--- |
| 场景表示 | 多为连续体积场 + 网络查询 | **显式** 高斯原语集合 |
| 渲染代价 | 每像素多采样 + 网络前向，传统实现往往较慢 | **光栅化 + 混合**，易做实时 |
| 编辑 / 剪枝 | 相对不直观 | 原语可增删、可局部调参 |
| 存储与显存 | 网络权重通常较小，但渲染慢 | 高质量常需 **大量原语**，显存压力大 |

二者并非简单「谁取代谁」：NeRF 及其加速变体仍在发展；3DGS 在 **交互浏览、VR/直播、部分 SLAM** 等低延迟场景优势明显。

## 7. 机器人与具身智能中的动机

传统机器人视觉常在 **稀疏几何**（快但欠表达）与 **高保真重建**（表达强但慢）之间取舍。3DGS 提供了一条 **照片级外观 + 实时渲染** 的路径，典型结合方式包括：

### 7.1 SLAM / 建图

将 3DGS 与位姿估计、关键帧管理结合，可在探索中维护 **可实时渲染的辐射场地图**，兼顾定位与稠密可视表达（具体系统各有 trade-off：是否全局 BA、是否语义层等）。

### 7.2 仿真与 Sim2Real

对真实环境快速重建 3DGS 场景后，可在高保真虚拟环境中做强化学习或数据增广，再迁移到真机，降低纯物理试错的成本与风险。

### 7.3 语义与操作

将 **CLIP 等语义特征** 与高斯或渲染图像对齐，可得到 **语义高斯（Semantic Splatting）** 一类表示，辅助抓取可操作部位识别、开放词汇查询等。

### 7.4 动态场景

**Dynamic / 4D Gaussian Splatting** 等为原语引入时间或运动模型，以缓解静态假设下的鬼影与模糊，服务动态环境中的感知与预测（仍属活跃研究方向）。

## 8. 工程与理论局限（简要）

- **显存与规模**：百万级以上高斯对 **边缘端 GPU** 不友好，压缩、蒸馏、层次 LOD 等是常见课题。
- **物理与规划**：高斯是软核，**碰撞、接触力学** 往往需要显式表面（Mesh）或 SDF；从高斯场提取可靠几何仍在探索中。
- **光照与材质**：复杂全局光照、强镜面与透明物体仍是难点，扩展模型（分解、物理先验）在持续发表。

## 9. 小结

3DGS 把新视角合成从「射线–网络」为主，推进到 **可微 2D splatting + 显式原语优化**，在实时性与视觉质量上改变了产品与研究中的默认选项之一；与机器人结合时，价值主要在 **地图表达、仿真与语义层**，落地仍需正视 **显存、几何提取与动态场景** 等约束。
